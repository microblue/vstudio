# VStudio — 产品概述

## 1. 产品定位

VStudio 是一个 **AI 驱动的电影短剧创作平台**（Web），将 AI 短剧制作 pipeline 产品化，为创作者提供从剧本到成片的全流程可视化工作台。

**一句话描述：** 用 AI 把你的故事变成电影。

## 2. 目标用户

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| 独立创作者 | 有故事创意但缺乏影视制作能力 | 低门槛、端到端出片 |
| 小型工作室 | 2-5人团队，需要快速出原型/概念片 | 效率、协作、成本控制 |
| 内容创业者 | 短视频/短剧赛道，需要批量化内容生产 | 批量、模板化、快速迭代 |
| 影视专业人士 | 导演/编剧/美术，用于 previz（预可视化） | 专业工作流、精细控制 |

## 3. 核心价值

1. **全流程覆盖** — 剧本 → 资产 → 分镜 → 画面 → 视频 → 音频 → 成片，一站式完成
2. **AI 驱动但人工可控** — 每个环节 AI 自动生成 + 人工审核/调整，不是黑盒
3. **专业级输出** — 1920×1080、24fps、H.264，直接可发布的成品质量
4. **低成本** — 本地 GPU 或云端 API，无需专业影视设备和团队
5. **可复现** — 所有参数、种子、提示词全部记录，任何结果都可重现和迭代

## 4. 核心概念

### 4.1 项目（Project）

一个创作项目，对应一部短剧或系列剧。包含多集（Episode）。

### 4.2 剧集（Episode）

一集内容，是基本的生产单元。从 script → final.mp4 的完整 pipeline。

### 4.3 资产（Asset）

项目级别的可复用元素：
- **角色（Character）** — 外貌、服装、性格描述 + 参考图
- **场景（Location）** — 环境、氛围、色彩 + 参考图
- **道具（Prop）** — 关键物品描述 + 参考图
- **风格圣经（Style Bible）** — 世界观、视觉风格、镜头语言规范

### 4.4 镜头（Shot）

一个连续画面片段，是视频生产的最小单元。包含：
- 景别/运镜（camera）
- 画面描述（action）
- 视觉提示词（prompt_visual）
- 运动提示词（prompt_motion）
- 台词（dialogue）
- 时长、转场、速度等参数

### 4.5 关键帧（Keyframe）

每个镜头内的关键画面节点。用于 T2I 生成静态图 → I2V 生成视频。

## 5. 产品形态

- **Web 应用**（SPA），浏览器端操作
- **后端服务** 编排 AI 任务（ComfyUI、LLM、TTS 等）
- **异步任务系统** 处理长时间的生成任务
- **文件存储** 管理生成的图片、视频、音频素材

## 6. 竞品参考

| 产品 | 特点 | VStudio 差异 |
|------|------|-------------|
| Runway | AI 视频生成工具 | VStudio 是全流程，不只是单个工具 |
| Pika | 文/图生视频 | VStudio 提供完整的剧本→成片工作流 |
| 可灵/即梦 | 国产 AI 视频生成 | VStudio 面向专业短剧制作，不是单条视频 |
| Final Cut / Premiere | 传统剪辑软件 | VStudio 侧重 AI 生成而非手动剪辑 |

## 7. MVP 范围

第一版聚焦：

✅ **包含：**
- 单用户模式（暂不做多用户协作）
- 完整的剧本→成片 pipeline
- 本地 ComfyUI 作为图片/视频生成后端
- LLM API（Claude/OpenAI）用于剧本解析和分镜生成
- TTS API（Fish Audio / Edge TTS）用于配音
- 基本的项目/剧集管理
- 资产库（角色、场景、道具）管理
- 镜头编辑器（可视化编辑 shots.json）
- 关键帧预览和选择
- 视频预览和导出

❌ **不包含（后续版本）：**
- 多用户/团队协作
- 云端 GPU 渲染
- 视频内容审核
- 支付/会员体系
- 模板市场
- 移动端适配
